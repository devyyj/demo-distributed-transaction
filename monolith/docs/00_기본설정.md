# monolith 프로젝트

## 디렉토리 구성

- 레이어드 아키텍처를 적용하여 디렉토리를 구성합니다.
- 컨트롤러, 서비스, 레포지토리 등의 레이어를 명확히 구분하여 패키지를 구성합니다.
- 도메인별 패키지를 구성합니다.
- 컨트롤러 -> 서비스 -> 레포지토리 순으로 의존성을 가지도록 구현합니다.
- 하위 레이어는 상위 레이어의 DTO를 참조하지 않습니다. 예를 들어 서비스 레이어는 컨트롤러 레이어의 DTO를 참조하지 않습니다.
- 상위 레이어가 하위 레이어로 데이터를 전달할 때는 하위 레이어의 DTO로 변환하여 전달합니다.
- 컨트롤러에서 정의한 Request DTO는 컨트롤러 레이어에서만 사용하며, 서비스 레이어로는 필요한 데이터만 추출하거나 별도의 서비스용 DTO(Command 등)로 변환하여 전달합니다.
- 각 레이어는 자신만의 데이터 객체(DTO)를 정의하여 사용합니다.

## 예외 처리
- 예외는 글로벌 예외 처리 클래스를 만들어서 처리합니다.
- 각 도메인에서 발생할 수 있는 예외를 정의하고, 글로벌 예외 처리 클래스에서 해당 예외를 처리하도록 구현합니다.
- 예외 발생 시 적절한 HTTP 상태 코드와 메시지를 반환하도록 구현합니다.

## 데이터 베이스

- docker compose 를 사용하여 mariadb 컨테이너를 실행합니다.
- yml 파일은 monolith 프로젝트의 루트 디렉토리에 위치한다.
- 컨테이너의 데이터 베이스 정보를 application.yml 파일에 작성한다.
