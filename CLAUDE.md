# 기본 규칙
- 모든 작업은 Best practices 에 따라 수행합니다.

# 코드 작성 규칙
- 모든 코드는 가독성을 고려하여 간결하게 작성합니다.
- 코드 내 주석은 명확하고 간결하게 작성하여 코드의 의도를 설명합니다.
- 변수명과 함수명은 의미를 명확히 전달할 수 있도록 작성합니다.
- 불필요한 코드나 중복된 코드는 제거하여 코드의 효율성을 높입니다.
- 코드 작성 시 일관된 스타일을 유지하여 가독성을 향상시킵니다.

# 프로젝트 구조
- 프로젝트는 레이어드 아키텍처 기반으로 구성합니다.
- 컨트롤러, 서비스, 리포지토리 레이어로 나누어 책임을 분리합니다.
- 각 레이어에서 사용하는 DTO(Data Transfer Object)를 정의하여 데이터 전달을 명확히 합니다.
- DTO 데이터 전환은 상위 레이어에서 수행하여 레이어 간의 의존성을 줄입니다.

# 데이터 베이스
