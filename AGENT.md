# 기본 규칙
- 모든 작업은 Best practices 에 따라 수행합니다.

# 코드 작성 규칙
- 모든 코드는 가독성을 고려하여 간결하게 작성합니다.
- 코드 내 주석은 명확하고 간결하게 작성하여 코드의 의도를 설명합니다.
- 변수명과 함수명은 의미를 명확히 전달할 수 있도록 작성합니다.
- 불필요한 코드나 중복된 코드는 제거하여 코드의 효율성을 높입니다.
- 코드 작성 시 일관된 스타일을 유지하여 가독성을 향상시킵니다.
- 기능의 흐름을 잘 파악할 수 있도록 로깅합니다.
- 옵션을 설정할 경우 appliction.yml 파일 설정을 우선합니다.
- lombok 라이브러리를 적극적으로 사용합니다.

# 프로젝트 구조
- 프로젝트는 레이어드 아키텍처 기반으로 구성합니다.
- 컨트롤러, 서비스, 리포지토리 레이어로 나누어 책임을 분리합니다.
- 각 레이어에서 사용하는 DTO(Data Transfer Object)를 정의하여 데이터 전달을 명확히 합니다.
- 하위 레이어는 상위 레이어가 사용하는 DTO를 참조하지 않습니다.
- 상위 레이어는 하위 레이어가 필요한 DTO에 맞추어서 데이터를 변환합니다. 

# 예외 처리
- 예외는 명확하게 정의된 커스텀 예외 클래스를 사용하여 처리합니다.
- 예외 메시지는 사용자에게 이해하기 쉬운 형태로 작성합니다.
- 예외 발생 시 적절한 HTTP 상태 코드를 반환하여 클라이언트가 상황을 명확히 알 수 있도록 합니다.
- 예외 처리는 전역적으로 관리하여 코드의 중복을 줄이고 일관된 예외 처리를 보장합니다.
