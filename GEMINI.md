# 기본 지침

## 필수 지침

- 커맨드 명령은 파워쉘 기준으로 실행합니다.
- 모든 응답은 한국어를 사용해야 합니다.
- 모든 코드는 최대한 간결해야 하며, 주석은 상세해야 합니다.
- 클린 코드 원칙을 준수해야 합니다.
- 모든 기능은 TDD(Test-Driven Development) 방식으로 개발되어야 합니다.-
- 문제가 발생할 경우 문제가 발생한 원인을 파악하고 유사한 문제가 발생하지 않도록 스스로 전체 프로세스를 점검하고 개선해야 합니다.

## 명령 실행 전

- 명령을 시작하기 전에, 사용자가 제공한 입력을 분석하여 명령의 목적과 요구 사항을 이해해야 합니다.
- 요구사항을 체크 리스트로 만들어서 사용자에게 보여주고, 사용자가 요구사항을 확인할 수 있도록 해야 합니다.

## 명령 실행 후

- 명령이 완료된 후, 결과를 사용자에게 명확하게 전달해야 합니다.
- 명령 시작 전에 작성했던 체크리스트를 모두 수행했는 지 확인해야 합니다.
- 모든 체크리스트를 수행하지 못하면 완료를 보고 하지 않습니다.
- 명령을 마치면 포매터, 린터, 테스트를 실행하여 코드의 품질을 보장해야 합니다.
